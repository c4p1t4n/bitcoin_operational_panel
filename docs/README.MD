O projeto é o Bitcoin Operations Panel — um painel de monitoramento e gestão de eventos da rede Bitcoin, onde cada coisa que acontece no node (novo bloco, spike de fee, tx suspeita, peer desconectado) vira um evento de domínio que é persistido, processado por regras e exibido em tempo real com audit trail completo.
O que torna esse projeto único é que o Bitcoin node resolve o problema mais chato de projetos de estudo: a fonte de dados. Você não escreve seed, não faz mock de preço, não simula nada. O mempool está emitindo eventos reais agora. Isso cria pressão real no backpressure do WebSocket, no rate do poller, no volume do event store — exatamente o que faz os patterns fazerem sentido de verdade em vez de parecerem teoria.

No backend, o BitcoinRPCAdapter encapsula o node como um Adapter pattern puro — o resto da aplicação não sabe que existe um HTTP RPC embaixo. O MempoolPoller emite eventos de domínio (MemPoolFeeSpike, NewBlockMined, LargeTransactionDetected) que vão para um Command bus. Cada command gera domain events que são persistidos no Postgres append-only e publicados no Redis simultaneamente.
